# tg-proxy — Telegram MTProto Proxy с ботом для контроля доступа

## Идея
Личный MTProto прокси для Telegram с контролем доступа через бота.
Каждому пользователю выдаётся уникальный secret на каждое устройство (1 secret = 1 сессия).
Поделился — сам вылетел.

## Архитектура

```
Пользователь → Telegram Bot → Админ одобряет → Генерация secret → mtg proxy
```

Компоненты:
- **mtg** — MTProto proxy сервер (fake-TLS, выглядит как обычный HTTPS)
- **tg-proxy** — Go бот, управляет пользователями и секретами
- **SQLite** — хранилище пользователей и секретов

## Команды бота

Пользователи:
- `/start` — запросить доступ
- `/more Ноутбук` — запросить доп. сессию для устройства
- `/my` — список своих прокси-ссылок

Админ:
- Inline-кнопки одобрения/отклонения в уведомлениях
- `/users` — список всех пользователей
- `/revoke @username` — отозвать все сессии
- `/kick @username` — забанить

## Деплой

ОС: **Ubuntu 24.04** на VPS

### Первый раз

1. Заказать VPS, поставить Ubuntu 24.04
2. Создать бота через @BotFather, получить токен
3. Узнать свой Telegram ID через @userinfobot
4. На маке:
   ```bash
   # Поменять SERVER в Makefile на IP сервера
   make setup    # установит mtg на сервер
   make deploy   # соберёт и зальёт бота
   ```
5. На сервере:
   ```bash
   cp /etc/tg-proxy/env.example /etc/tg-proxy/env
   nano /etc/tg-proxy/env   # заполнить BOT_TOKEN, ADMIN_ID, SERVER_HOST
   systemctl enable tg-proxy mtg
   systemctl start tg-proxy
   ```

### Обновление кода

```bash
make quick   # собрать + залить + рестарт
```

## Защита от шаринга

Основная: 1 secret = 1 concurrent connection (ограничение mtg + если кто-то подключился с твоим secret — тебя выбьет).

Дополнительная (на будущее): мониторинг через mtg Prometheus метрики — алерт если с одного secret'а подключаются с 3+ разных IP за сутки.

